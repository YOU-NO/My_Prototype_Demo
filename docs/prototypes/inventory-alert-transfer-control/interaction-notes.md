# 库存预警与调拨补货控制原型说明

## 原型目的

本原型用于说明“库存预警与调拨补货控制”的前端交互方案，不连接真实接口，不修改生产代码。原型重点参考现有“库存列表/汇总库存”页面的企业后台布局、筛选区、`ProTable` 列表、表头按钮和弹窗模式。

## 页面清单

- 汇总库存/库存列表：保留原有总库存字段，展示新增当前库存、库存状态、库存上限、超阈值数量、导入库存阈值、编辑阈值和风险详情。
- 库存阈值导入弹窗：展示模板下载、文件上传、导入校验说明和失败数据下载入口。
- 编辑库存上限弹窗：模拟单条阈值实时调整。
- 调拨提交风险弹窗：模拟备货计划提交调拨发货或合并提交调拨发货时的库存水位预警，用户确认风险后可继续推单。

## 关键交互

1. 点击“搜索”：
   - 根据筛选条件刷新表格。
   - 原型通过提示条模拟查询完成。

2. 点击“只看库存过高”：
   - 自动将库存状态筛选为“库存过高”。
   - 表格突出红色风险行。

3. 点击“导入库存阈值”：
   - 打开导入弹窗。
   - 支持“下载模板”和“选择文件”演示。
   - 点击“确认导入”后模拟成功提示，并关闭弹窗。

4. 点击表格行“编辑阈值”：
   - 打开阈值编辑弹窗。
   - 展示仓库、SKU、当前库存和原库存上限。
   - 保存后模拟刷新列表并提示“阈值已更新，库存状态已重新计算”。

5. 点击表格行“查看风险”：
   - 打开风险详情弹窗。
   - 展示当前库存、可用库存、在途库存、阈值、超出数量和建议动作。

6. 点击“模拟提交调拨”：
   - 打开调拨水位校验结果弹窗。
   - 展示库存水位过高预警和超阈值 SKU 明细。
   - 每条超阈值明细行提供“调整阈值”操作，用于进入该 SKU 的阈值编辑弹窗。
   - 点击“继续推单”后，模拟允许用户确认风险并继续推单。

## 字段和 mock 数据说明

- `inventoryRows`：库存列表 mock 数据，包含仓库、SKU、可用库存、在途库存、库存上限、库存状态等。
- `transferRiskItems`：调拨提交时的超阈值 SKU 明细，包含当前库存、本次调拨入库数量、调拨后库存、阈值和超出数量。
- `summary`：顶部统计卡片数据。
- `standardTemplateColumns`：库存阈值导入模板字段。

## 产品假设

- “汇总库存”对应当前代码中的 `xzy-erp-ui/src/views/wms/inventory/index.vue` 库存列表页面。
- 本期只处理库存过高预警，不处理库存不足和库存结构异常。
- 保留现有总库存字段，口径为“总库存 = 可用库存 + 锁定库存”。
- 当前库存按“可用库存 + 在途库存”计算，在途库存默认包含调拨在途，是否包含采购在途需业务确认。
- 库存阈值按仓库 x SKU 维护，可通过 Excel 或系统导入，也支持页面单条实时调整。
- 调拨提交侧按“当前库存 + 本次调拨入库数量”校验；若超过库存阈值，则提示库存过高风险，但允许用户确认后继续推单。

## 前端实现提示

- 库存列表建议在 `xzy-erp-ui/src/views/wms/inventory/index.vue` 的 `columns` 中扩展水位字段，并延续现有 `ProTable`、表头按钮、行级 link button、弹窗风格。
- 库存状态可使用 Element Plus `el-tag`：`OVER_LIMIT` 使用 `danger`，`NORMAL` 使用默认或 `info`，`NO_STANDARD` 使用 `warning`。
- 导入库存阈值可复用项目中已有 `ImportExcel` 组件模式，支持模板下载和错误数据导出。
- 调拨提交校验建议在 `formDialog.vue` 的 `handleSubmitAllot` 和 `combineAllotDialog.vue` 的 `handleSave` 中接入结果弹窗，但正式拦截必须以后端接口返回为准。
- 风险弹窗建议独立组件化，例如 `InventoryWaterLevelRiskDialog.vue`，供单个提交和合并提交复用。
